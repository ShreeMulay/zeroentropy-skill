# Change: v1.1.5 Review Hardening

## Status
completed

## Description

Remediated the full-project review findings discovered after v1.1.4. The change hardens the OpenCode plugin around cancellation, mutation retry semantics, and local validation; makes CI enforce the repository-level regression suite; packages functional release artifacts; and cleans stale project/spec hygiene.

## Impact

- `plugin/src/index.ts` — abort-aware retries, mutation-safe retry handling, stricter metadata/page/embed validation.
- `plugin/tests/` — red-first coverage for cancellation, mutation ambiguity, validation, metadata, and permission denial.
- `.woodpecker.yml`, `.github/workflows/ci.yml`, `.github/workflows/release.yml` — full regression tests and functional release packaging.
- `tests/test_v114_review_remediation.py` — repository validation for release workflow, CI, docs, OpenSpec, changelog, gitignore, and version alignment.
- `README.md`, `SKILL.md`, `CHANGELOG.md`, `openspec/project.md`, `.gitignore` — documentation and hygiene updates.
- `openspec/changes/archive/` — archived completed/stale changes.
- Version metadata — bumped to `1.1.5`.
