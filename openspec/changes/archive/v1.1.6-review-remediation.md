## Findings Remediated

### Medium
1. **SKILL.md error-handling drift** — claimed "All plugin tools implement automatic retry with exponential backoff"; since v1.1.5 mutating tools intentionally fail fast. Rewrote §Error Handling to document read-retry vs mutation fail-fast, abort handling, and the delete permission gate.
2. **plugin/README error-handling drift** — claimed the plugin "Validates `list:` prefix" for filters (false; filters pass through unvalidated). Rewrote with accurate semantics plus an explicit note that query filters are not rewritten.
3. **Woodpecker parity gap** — added `lint-docs` step (node:22: `npm ci`, `npm run lint`, `npm run typecheck`) so primary Forgejo CI matches GitHub mirror lint/typecheck coverage.

### Low
4. **`sleep()` abort-listener leak** — backoff timer now removes its abort listener on normal resolve.
5. **Batch abort under-reporting** — aborted batches report `skipped_count`; `success + failed + skipped === documents.length` always reconciles.
6. **Dead SDK fallbacks** — removed nonexistent `collections.create` / `collections.remove` probes; direct typed `add`/`delete` calls.
7. **Version-pinned release-docs test** — now derives the expected tarball name and tag from `skill.json` version.
8. **Misnamed test module** — `tests/test_v114_review_remediation.py` → `tests/test_repo_validation.py`.
9. **Dep drift** — `@opencode-ai/plugin` dev/peer → `^1.17.3` (installed 1.17.3; build/tests green).
