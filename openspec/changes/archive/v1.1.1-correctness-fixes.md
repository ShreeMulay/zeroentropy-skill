# Change: v1.1.1 Plugin Correctness & Consistency Fixes

## Status
completed

## Description
Address review findings from comprehensive codebase review. Fixes a critical test/SDK mismatch in the status tool, removes a misleading no-op parameter, aligns versions to 1.1.0, adds batch pages parity, and completes plugin documentation.

## Motivation
A full codebase review (plus live API testing) surfaced one critical bug and several consistency issues:
1. `zeroentropy_status` unit test mocks `documents.status`, which does not exist in the SDK. The real path (`documents.getInfo`, envelope `{ document: { index_status } }`) is untested.
2. `zeroentropy_index` exposes an `overwrite` arg that is silently dropped (live API rejects it).
3. Manifest versions (`1.0.0`) lag the committed v1.1.0 feature work.
4. `zeroentropy_batch` lacks `pages[]` support that `zeroentropy_index` has.
5. `plugin/README.md` documents only 4 of 9 tools; `.beads` runtime artifacts may be tracked.

## Impact
- `plugin/src/index.ts` — remove `overwrite` from index schema; fix 409 suggestion text; add `pages` to batch schema
- `plugin/tests/index.test.ts` — mock `getInfo` with real envelope; add `not_indexed` normalization test; batch pages test
- `plugin/package.json`, `skill.json`, `SKILL.md` — bump to 1.1.0
- `plugin/README.md` — document all 9 tools
- `.gitignore` — ignore `.beads` runtime artifacts
- `CHANGELOG.md` — add v1.1.1 entry
